# 多模态文物RAG系统数据集需求说明

## 一、项目目标

构建一个基于 **Spring AI + RAG + CN-CLIP**
的多模态文物知识问答系统，实现：

-   文本问答（RAG）
-   以图搜文物（图像检索）
-   图文联合检索
-   文物知识解释生成

------------------------------------------------------------------------

## 二、数据集总体要求

### 1. 数据类型

每条数据必须包含：

-   文本信息（用于RAG）
-   图片（用于CN-CLIP）
-   元数据（用于过滤与评测）

------------------------------------------------------------------------

### 2. 数据单位

一个文物 = 一条数据（不可混合）

------------------------------------------------------------------------

## 三、数据结构规范

``` json
{
  "id": "artifact_001",
  "name": "何尊",
  "era": "西周早期",
  "category": "青铜器",
  "museum": "中国国家博物馆",
  "origin": "陕西宝鸡",
  "description": "何尊是西周早期青铜礼器...",
  "features": ["口沿外撇","腹部较深","圈足"],
  "value_points": ["铭文出现最早“中国”","反映西周政治结构"],
  "image": "hezun.jpg",
  "image_caption": "青铜器，器型厚重，内壁有铭文",
  "keywords": ["何尊","青铜器","中国起源","西周"]
}
```

------------------------------------------------------------------------

## 四、图片要求

-   推荐 JPG 格式
-   文物主体清晰
-   无水印
-   居中

------------------------------------------------------------------------

## 五、文本要求

-   200\~500字
-   包含描述、特征、价值

------------------------------------------------------------------------

## 六、元数据要求

必须包含：

-   era
-   category
-   keywords

------------------------------------------------------------------------

## 七、数据规模建议

-   第一阶段：20\~50条
-   第二阶段：100\~300条

------------------------------------------------------------------------

## 八、评测数据

``` json
{
  "question": "最早出现中国一词的文物是什么？",
  "answer": "何尊",
  "expected_doc": "artifact_001"
}
```

------------------------------------------------------------------------

## 九、核心原则

数据不是文章，而是可检索知识单元
